# Mangarr Support

Mangarr is maintained by one developer. Support is best effort, with priority
given to data loss, security, upgrade failures, broken imports, and regressions
in supported integrations.

## Where to Ask

- Use [GitHub Discussions](https://github.com/Kha-kis/manga-arr/discussions)
  for installation, configuration, metadata-provider, and workflow questions.
- Use the bug report form for reproducible defects.
- Use the feature request form for scoped product proposals.
- Follow `SECURITY.md` for vulnerabilities. Do not open a public security issue.

## Include These Details

- Mangarr version and image tag or digest.
- Deployment method, host platform, and container architecture.
- The affected indexer, download client, metadata provider, and archive format.
- Exact reproduction steps, expected behavior, and actual behavior.
- Relevant logs with credentials, cookies, API keys, tracker URLs, and private
  filesystem paths removed.
- Whether the problem reproduces after a restart and whether a backup exists.

The project supports the current stable release and the current public release
candidate as described in `SECURITY.md`. Unsupported local modifications should
be reduced to a clean reproduction before they can be investigated reliably.
